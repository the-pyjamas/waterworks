# Contributing

Thank you for your interest in contributing to this project.

We welcome contributions of all sizes, including bug reports, documentation improvements, feature suggestions, and code contributions.

## Reporting Issues

Before opening a new issue, please check whether it has already been reported.

When reporting a bug, include:

* A clear description of the problem.
* Steps to reproduce it.
* Expected behavior.
* Actual behavior.
* Environment details when applicable.

## Pull Requests

Before submitting a pull request:

* Fork the repository.
* Create a new branch for your work.
* Keep commits focused and meaningful.
* Follow the existing project structure and coding style.
* Ensure your changes do not break existing functionality.

When your work is ready, open a pull request and provide a clear description of the changes.

## Code Style

Please follow the existing conventions used throughout the project.

Consistency is preferred over personal style.

## Discussions

For feature ideas or architectural changes, open an issue before starting implementation. This helps avoid duplicated work and ensures the proposal aligns with the project's direction.

## Code of Conduct

Be respectful and constructive in all discussions and contributions.

Thank you for helping improve this project.
